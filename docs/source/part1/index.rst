Chapitre 1 | Introduction aux méthodologies de développement logiciel
*************************************************************************************************

Objectifs
=========

À l'issue de ce module, chaque étudiant.e sera capable de :

* Définir les concepts fondamentaux des méthodologies de développement logiciel
* Expliquer les principales méthodologies : Cascade, V, Agile, Scrum et Spirale
* Comparer les avantages et inconvénients de chaque méthodologie
* Identifier la méthodologie appropriée selon le contexte d'un projet donné
* Expliquer les rôles et responsabilités dans chaque méthodologie

Notes
""""""
- Une méthodologie de développement logiciel est un ensemble structuré de pratiques, processus et principes pour organiser le développement d'un logiciel
- Les méthodologies répondent à des questions essentielles : Comment planifier ? Comment organiser le travail ? Comment gérer les changements ? Comment assurer la qualité ?
- Pourquoi utiliser une méthodologie : réduire les risques, améliorer la communication, garantir la qualité, respecter les délais et budgets

**Les principales méthodologies :**

1. **Méthodologie Cascade (Waterfall)**
   
   * Approche séquentielle linéaire
   * Phases : Analyse → Conception → Développement → Tests → Déploiement → Maintenance
   * Avantages : Simple, structurée, documentation complète
   * Inconvénients : Rigide, peu adaptable aux changements, feedback tardif
   * Contexte d'application : Projets avec exigences stables et bien définies

2. **Modèle en V**
   
   * Extension du modèle Cascade avec phases de validation
   * Chaque phase de développement correspond à une phase de test
   * Avantages : Tests planifiés dès le début, meilleure traçabilité
   * Inconvénients : Rigide, coûteux en cas de modifications
   * Contexte d'application : Projets critiques nécessitant validation rigoureuse

3. **Méthodologie Agile**
   
   * Approche itérative et incrémentale
   * Principes : Collaboration, adaptation au changement, livraisons fréquentes
   * Manifeste Agile : Individus et interactions > Processus et outils
   * Avantages : Flexibilité, feedback continu, satisfaction client
   * Inconvénients : Moins de documentation, nécessite discipline
   * Contexte d'application : Projets avec exigences évolutives

4. **Scrum**
   
   * Framework Agile avec rôles, événements et artefacts définis
   * Rôles : Product Owner, Scrum Master, Équipe de développement
   * Événements : Sprint, Daily Scrum, Sprint Planning, Sprint Review, Sprint Retrospective
   * Artefacts : Product Backlog, Sprint Backlog, Increment
   * Avantages : Transparence, inspection, adaptation rapide
   * Contexte d'application : Équipes auto-organisées, projets complexes

5. **Modèle en Spirale**
   
   * Approche itérative combinant développement et gestion des risques
   * Phases répétées : Planification → Analyse des risques → Développement → Évaluation
   * Avantages : Gestion proactive des risques, prototypage
   * Inconvénients : Complexe, coûteux, nécessite expertise
   * Contexte d'application : Projets à haut risque, grands projets innovants
